# Repository Operating Rules

These rules capture the owner's standing preferences for work in this repository.

- In final summaries, state what changed, why it matters, and what remains to do.
- Keep `README.md` accurate as workflows, setup steps, scripts, secrets, or operating assumptions change.
- Do not commit or push changes unless the owner explicitly asks for that Git action.
- Do not commit local secrets, `.env`, virtual environments, caches, or generated local-only artifacts.
- Preserve the approval rule: automation sends only email drafts explicitly marked `approved`.
